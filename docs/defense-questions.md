# Подготовка к защите: краткие ответы

1. `Comparable` и `Comparator`:
`Comparable` задаёт естественный порядок внутри класса, `Comparator` задаёт внешний порядок сортировки.

2. Категории коллекций:
`List` — упорядоченная последовательность, `Set` — уникальные элементы, `Queue` — очередь обработки, `Map` — пары ключ-значение.

3. Параметризованные типы:
Generics повышают типобезопасность и убирают касты, wildcard (`? extends`, `? super`) задаёт ограниченный контракт.

4. Классы-оболочки:
Позволяют использовать примитивы как объекты, поддерживают utility-методы, участвуют в автоупаковке/распаковке.

5. Потоки ввода-вывода:
Байтовые потоки работают с `byte`, символьные — с `char`, цепочки потоков позволяют комбинировать поведение.

6. Работа с файлами:
В проекте чтение через `FileReader`, запись через `FileOutputStream`, ошибки I/O обрабатываются без аварийного завершения.

7. `java.nio`:
Современный API для путей, файловых операций и неблокирующего I/O, в проекте применяется для нормализации пути скрипта.

8. `javadoc`:
Инструмент генерации документации из комментариев `/** ... */`, используется для описания назначений классов.
